GatewayControllerCreatorOptions

Source:
  • controllers/utils/gateway.js, line 138
Properties:
Name Type Attributes Description
serviceName string

The name of the creator will use to register the controller in the container. No, this is not a typo. The creator will register the controller so other services can access the endpointsForAPIClient getter. The service will be available after the app routes are mounted. If this is overwritten, the creator will ensure that the name ends with Gateway; and if overwritten, but it doesn't include Gateway at the end, and no configurationSetting was defined, the creator will use the custom name (without Gatway) for configurationSetting. Default 'apiGeteway'.
helperServiceName string

The name of the helper service the creator will try to obtain from the container. If serviceName is overwritten, the default for this will be ${serviceName}Helper. Default: 'apiGatewayHelper'.
configurationSetting string

The name of the configuration setting where the gateway configuration is stored. If not overwritten, check the description of serviceName to understand which will be its default value. Default 'api'.
gatewayClass Class.<GatewayController>

The class the creator will instantiate. Similar to APIClient, this allows for extra customization in cases you may need multiple gateways. Default GatewayController.
middlewares GatewayControllerMiddlewaresFn <nullable>

This function can be used to add custom middlewares on the gateway routes. If implemented, it must return a list of middlewares when executed. Default null.

This are the options sent to the controller creator that instantiates GatewayController. They're basically the same as GatewayControllerOptions but with a couple of extra ones.

GatewayControllerOptions

Source:
  • controllers/utils/gateway.js, line 115
Properties:
Name Type Description
root string

This is really a helper for when the gateway is used with an API client. The idea is that, by default, the routes are mounted on the controller route, but with this option, you can specify another sub path. For example: The controller is mounted on /routes, if you set root to gateway, all the routes will be on /routes/gateway. This become important (and useful) when you get the API client configuration (with endpointsForAPIClient): The url will be the controller route, but all the endpoints will be modified and prefixed with the root. Default ''.
configurationSetting string

This is another option for when the gateway is used with an API client. When calling endpointsForAPIClient, all the endpoints will be wrapped inside an object named after this option. For example: { url: '...', endpoints: { api: { ... } } }. Default api.
headers GatewayControllerHeadersOptions

The options for how the gateway will handle the headers from the requests and the responses.

The options to configure how the gateway will manage the requests and the responses.
